3 Compensation Tracking ④ NEEDS DECISION
How deal_expansion_type drives comp — proposed values
Set by RM on each new Renewal Deal · comp rules TBD
⚠ Compensation structure not confirmed. Values below are proposed. Need to align with Finance on bonus/chargeback rules before finalizing.
None

Standard Renewal

No upsell history or never upsold. Standard renewal comp only.

Upsell-New

New Upsell Added

First time adding a module to this account. New upsell bonus triggered.

Upsell-Retained

Upsell Kept at Renewal

Had upsell last year, renewing it. New Upsell Deal auto-spins up. Retention bonus.

Upsell-Dropped

Upsell Removed

Had upsell, not renewing it. No Upsell Deal created. Chargeback / reduced comp.

4 Automation Workflows to Build
4 HubSpot workflow triggers required
All dependent on decisions ①–④ being resolved first
1

Upsell Deal Creation

When Renewal Deal's deal_expansion_type is set to Upsell-New or Upsell-Retained → create Upsell Deal in Upsell Pipeline, associate back to Renewal Deal.

Trigger: deal_expansion_type IN [Upsell-New, Upsell-Retained]
2

ARR Rollup

When Upsell Deal amount is set or deal reaches Closed Won → sum all associated Upsell Deal amounts → write to Renewal Deal's upsell_arr.

Trigger: Upsell Deal.amount changes OR dealstage = Closed Won
3

Total Renewal Calculation

When amount or upsell_arr changes on Renewal Deal → recalculate total_next_renewal_arr = amount + upsell_arr.

Trigger: Renewal Deal.amount or .upsell_arr changes
4

Next Renewal Creation & previous_year_arr Sync ④ DECIDE

Copy total_next_renewal_arr from closing renewal into previous_year_arr on the new renewal deal. Starting amount of next renewal = total_next_renewal_arr.

→ Is next renewal created automatically (date trigger, 90 days out) or manually by RM?

Trigger: TBD — date-based auto or manual RM action?

Property Name — Blocker

Recommendation: extend existing — fewer properties, already used for reporting.

Default Upsell Stage
The Upsell Pipeline's first stage is 50% Demo Held. Is this the correct stage for auto-created upsell deals, or should they land at a different stage (e.g. a new "Identified" stage)?
Recommendation: confirm 50% Demo Held — or rename it to better reflect the upsell motion.
③
contract_products Location
Should contract_products live on the Renewal Deal only (simpler), or on both Renewal Deal and Upsell Deal (per-module upsell tracking)?
Recommendation: both — Upsell Deal captures what was added; Renewal Deal captures what's in the full contract.
